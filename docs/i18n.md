# Storm TUI Internationalization (i18n)

Storm TUI includes a built-in i18n system with locale definitions, string interpolation, pluralization, number formatting, and RTL direction support.

## Locale system

The `Locale` interface defines everything needed for a language:

```ts
interface Locale {
  /** ISO 639-1 code, e.g. "en", "ja", "ar" */
  readonly code: string;
  /** Text direction */
  readonly direction: "ltr" | "rtl";
  /** Number formatting */
  readonly numbers: NumberFormat;
  /** Full month names (January..December) */
  readonly months: readonly string[];
  /** Short month names (Jan..Dec) */
  readonly monthsShort: readonly string[];
  /** Full weekday names (Sunday..Saturday) */
  readonly weekdays: readonly string[];
  /** Short weekday names (Sun..Sat) */
  readonly weekdaysShort: readonly string[];
  /** Translatable UI strings keyed by dot-path */
  readonly strings: Readonly<Record<string, string>>;
  /** Plural rule for this locale (defaults to English rules if omitted) */
  readonly pluralRule?: PluralRule;
}
```

The `NumberFormat` sub-interface controls numeric display:

```ts
interface NumberFormat {
  readonly decimal: string;    // e.g. "." or ","
  readonly thousands: string;  // e.g. "," or "."
  readonly grouping: number;   // digits per group (typically 3)
}
```

The built-in `EN` locale is always available as the default:

```ts
import { EN } from "@orchetron/storm";
// EN.code === "en", EN.direction === "ltr"
// EN.numbers === { decimal: ".", thousands: ",", grouping: 3 }
```

## Translation

The `t()` function looks up a key in the locale's `strings` record and interpolates `{name}` placeholders. If the key is not found, it returns the key itself as a fallback.

```ts
import { t } from "@orchetron/storm";

const locale = {
  ...EN,
  strings: {
    "greeting": "Hello, {name}!",
    "status.connected": "Connected to {server} ({count} users)",
  },
};

t("greeting", locale, { name: "World" });
// => "Hello, World!"

t("status.connected", locale, { server: "us-east", count: 42 });
// => "Connected to us-east (42 users)"

t("missing.key", locale);
// => "missing.key"  (falls back to the key itself)
```

Signature:

```ts
function t(
  key: string,
  locale: Locale,
  params?: Record<string, string | number>,
): string;
```

## Pluralization

The `plural()` function selects the correct string form based on a count and the locale's plural rule. It looks up `${key}.${category}` in the strings, falling back to `${key}.other`, then to the raw count.

```ts
import { plural, EN } from "@orchetron/storm";

const messages = {
  "items.zero": "No items",
  "items.one": "{count} item",
  "items.other": "{count} items",
};

plural("items", 0, EN, messages);  // => "No items"
plural("items", 1, EN, messages);  // => "1 item"
plural("items", 5, EN, messages);  // => "5 items"
```

### Built-in plural rules

Storm TUI ships five plural rules covering the major language families:

| Rule | Languages | Categories used |
|------|-----------|----------------|
| `PLURAL_EN` | English | `one` (n=1), `other` |
| `PLURAL_FR` | French, Portuguese | `one` (n<=1), `other` |
| `PLURAL_AR` | Arabic | `zero`, `one`, `two`, `few` (3-10), `many` (11-99), `other` |
| `PLURAL_RU` | Russian, Polish | `one`, `few`, `many` |
| `PLURAL_JA` | Japanese, Chinese, Korean | `other` (no plural forms) |

All six categories are defined by the `PluralCategory` type: `"zero" | "one" | "two" | "few" | "many" | "other"`.

You can also define your own plural rule:

```ts
const PLURAL_CUSTOM: PluralRule = {
  select: (n) => {
    if (n === 0) return "zero";
    if (n === 1) return "one";
    return "other";
  },
};
```

## Number formatting

The `formatNumber()` function formats numbers using the locale's decimal separator, thousands separator, and grouping size:

```ts
import { formatNumber, EN } from "@orchetron/storm";

formatNumber(1234567.89, EN);   // => "1,234,567.89"
formatNumber(-42, EN);          // => "-42"
formatNumber(1000, EN);         // => "1,000"
```

With a European-style locale:

```ts
const DE: Locale = {
  ...EN,
  code: "de",
  numbers: { decimal: ",", thousands: ".", grouping: 3 },
};

formatNumber(1234567.89, DE);  // => "1.234.567,89"
```

## LocaleProvider

Wrap your app in `<LocaleProvider>` to set the locale for the entire subtree. Without it, all hooks fall back to the built-in `EN` locale.

```tsx
import { LocaleProvider, render } from "@orchetron/storm";
import { jaLocale } from "./locales/ja.js";

render(
  <LocaleProvider locale={jaLocale}>
    <App />
  </LocaleProvider>
);
```

The `useLocale()` hook reads the current locale from context:

```tsx
import { useLocale, t } from "@orchetron/storm";

function Greeting() {
  const locale = useLocale();
  return <Text>{t("hello", locale)}</Text>;
}
```

## Custom locales

### Defining a locale

Create a `Locale` object with all required fields:

```ts
import { EN, PLURAL_FR, type Locale } from "@orchetron/storm";

const FR: Locale = {
  code: "fr",
  direction: "ltr",
  pluralRule: PLURAL_FR,
  numbers: { decimal: ",", thousands: " ", grouping: 3 },
  months: [
    "janvier", "fevrier", "mars", "avril", "mai", "juin",
    "juillet", "aout", "septembre", "octobre", "novembre", "decembre",
  ],
  monthsShort: [
    "janv.", "fevr.", "mars", "avr.", "mai", "juin",
    "juil.", "aout", "sept.", "oct.", "nov.", "dec.",
  ],
  weekdays: [
    "dimanche", "lundi", "mardi", "mercredi",
    "jeudi", "vendredi", "samedi",
  ],
  weekdaysShort: ["dim.", "lun.", "mar.", "mer.", "jeu.", "ven.", "sam."],
  strings: {
    "greeting": "Bonjour, {name} !",
    "items.one": "{count} element",
    "items.other": "{count} elements",
  },
};
```

### Registering for lookup

Use `registerLocale()` to add a locale to the global registry so it can be retrieved by code:

```ts
import { registerLocale, getLocale, getRegisteredLocales } from "@orchetron/storm";

registerLocale(FR);

const locale = getLocale("fr");    // => FR locale object
const codes = getRegisteredLocales(); // => ["en", "fr"]
```

The registry is a simple `Map<string, Locale>`. The `EN` locale is pre-registered under `"en"`.

## Complete example

A multilingual greeting app that switches between English and Arabic:

```tsx
import {
  render,
  LocaleProvider,
  useLocale,
  t,
  plural,
  formatNumber,
  EN,
  PLURAL_AR,
  type Locale,
} from "@orchetron/storm";
import { useState } from "react";

const AR: Locale = {
  code: "ar",
  direction: "rtl",
  pluralRule: PLURAL_AR,
  numbers: { decimal: ".", thousands: ",", grouping: 3 },
  months: [
    "\u064a\u0646\u0627\u064a\u0631", "\u0641\u0628\u0631\u0627\u064a\u0631", "\u0645\u0627\u0631\u0633", "\u0623\u0628\u0631\u064a\u0644", "\u0645\u0627\u064a\u0648", "\u064a\u0648\u0646\u064a\u0648",
    "\u064a\u0648\u0644\u064a\u0648", "\u0623\u063a\u0633\u0637\u0633", "\u0633\u0628\u062a\u0645\u0628\u0631", "\u0623\u0643\u062a\u0648\u0628\u0631", "\u0646\u0648\u0641\u0645\u0628\u0631", "\u062f\u064a\u0633\u0645\u0628\u0631",
  ],
  monthsShort: [
    "\u064a\u0646\u0627", "\u0641\u0628\u0631", "\u0645\u0627\u0631", "\u0623\u0628\u0631", "\u0645\u0627\u064a", "\u064a\u0648\u0646",
    "\u064a\u0648\u0644", "\u0623\u063a\u0633", "\u0633\u0628\u062a", "\u0623\u0643\u062a", "\u0646\u0648\u0641", "\u062f\u064a\u0633",
  ],
  weekdays: [
    "\u0627\u0644\u0623\u062d\u062f", "\u0627\u0644\u0627\u062b\u0646\u064a\u0646", "\u0627\u0644\u062b\u0644\u0627\u062b\u0627\u0621", "\u0627\u0644\u0623\u0631\u0628\u0639\u0627\u0621",
    "\u0627\u0644\u062e\u0645\u064a\u0633", "\u0627\u0644\u062c\u0645\u0639\u0629", "\u0627\u0644\u0633\u0628\u062a",
  ],
  weekdaysShort: ["\u0623\u062d\u062f", "\u0627\u062b\u0646", "\u062b\u0644\u0627", "\u0623\u0631\u0628", "\u062e\u0645\u0633", "\u062c\u0645\u0639", "\u0633\u0628\u062a"],
  strings: {
    "greeting": "\u0645\u0631\u062d\u0628\u0627\u060c {name}!",
    "items.zero": "\u0644\u0627 \u0639\u0646\u0627\u0635\u0631",
    "items.one": "\u0639\u0646\u0635\u0631 \u0648\u0627\u062d\u062f",
    "items.two": "\u0639\u0646\u0635\u0631\u0627\u0646",
    "items.few": "{count} \u0639\u0646\u0627\u0635\u0631",
    "items.many": "{count} \u0639\u0646\u0635\u0631\u0627",
    "items.other": "{count} \u0639\u0646\u0635\u0631",
    "stats": "\u0627\u0644\u0645\u062c\u0645\u0648\u0639: {total}",
  },
};

function Dashboard() {
  const locale = useLocale();
  const dir = locale.direction;
  const itemCount = 7;
  const total = 1234567;

  return (
    <Box flexDirection="column">
      <Text>{t("greeting", locale, { name: "Storm" })}</Text>
      <Box flexDirection={dir === "rtl" ? "row-reverse" : "row"}>
        <Text>{plural("items", itemCount, locale)}</Text>
        <Text> | </Text>
        <Text>{t("stats", locale, { total: formatNumber(total, locale) })}</Text>
      </Box>
    </Box>
  );
}

function App() {
  const [locale, setLocale] = useState<Locale>(EN);

  return (
    <LocaleProvider locale={locale}>
      <Dashboard />
      <Box marginTop={1}>
        <Text>Press "e" for English, "a" for Arabic</Text>
      </Box>
    </LocaleProvider>
  );
}

render(<App />);
```
